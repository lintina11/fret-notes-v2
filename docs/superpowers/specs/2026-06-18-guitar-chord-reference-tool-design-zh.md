# 吉他和弦對照工具 — 設計文件
日期：2026-06-18

## 專案概述

一個以吉他為主入口的跨樂器和弦對照網頁工具。使用者透過點選虛擬指板來即時識別和弦，並同步顯示和弦名稱、組成音、音程結構與鋼琴鍵盤對照。

- **主要使用者：** 吉他手，樂理程度基礎到中階
- **主要裝置：** iPad（觸控優先），同時支援手機與桌機

---

## 技術棧

- **框架：** Nuxt 3（Vue 3 + TypeScript），靜態部署（`nuxt generate`）
- **部署平台：** Vercel 或 Netlify
- **樣式：** CSS 變數實現 light/dark 主題切換
- **未來擴充：** Supabase（使用者帳號 + 儲存自訂指型）— Phase 1 不實作

---

## Phase 1 範疇

### 包含功能
- 互動指板（點選格子新增/移除按壓點）
- 即時和弦識別
- 和弦結果顯示：和弦名稱、轉位（slash chord）、候選替代名稱、組成音、音程
- 鋼琴鍵盤對照（唯讀顯示）
- Light / Dark 模式切換
- 響應式佈局（iPad 橫式/直式、手機）

### 不包含（Phase 2+）
- 鋼琴 → 吉他方向（點鋼琴鍵推薦吉他指型）
- 使用者帳號與自訂指型儲存（Supabase）
- 其他調弦（Drop D、開放調弦等）
- Phase 1 以外的延伸和弦型

## 樂理資料層

### Phase 1 支援的和弦類型

| 類別 | 和弦型 |
|------|--------|
| 三和弦 | 大三、小三、減三、增三 |
| 七和弦 | maj7、m7、屬七（7）、m7b5 |
| 掛留和弦 | sus2、sus4 |
| 加音和弦 | add9、6、m6 |

### 識別邏輯

和弦識別採用**音程集合比對**，不用硬編表格。

流程：
1. 從選取的格子收集所有不重複的音級（0–11，忽略八度）
2. 以每個可能的音為根音，計算音程集合
3. 比對音程規則定義（例如 `[0, 3, 7]` = 小三和弦）
4. 排序候選結果，取第 1 名為主要和弦，最多顯示 2 個備選

**轉位（Slash chord）：** 若最低音不是根音，自動顯示為 `Am/E` 格式。

**無法識別：** 只顯示組成音，不強行命名。

### 檔案結構

```
core/music-theory/
├── notes.ts          # 12音音級、音名、等音對應
├── intervals.ts      # 音程計算
├── chord-rules.ts    # 音程集合 → 和弦名稱定義表
└── chord-detector.ts # 主識別函式：notes[] → ChordResult
```

樂理邏輯全部是純 TypeScript，不依賴任何 UI，可獨立測試。

## 元件設計

### `<Fretboard />` 互動指板

**規格：**
- 6 弦 × 12 格，預設顯示第 1–5 格（可捲動至第 12 格）
- 標準調弦：E2 A2 D3 G3 B3 E4
- 左側：格數編號
- 頂部：空弦指示圓圈

**空弦邏輯：**
- 該弦無按壓點 → 顯示空弦圓圈（演奏空弦）
- 該弦有按壓點 → 空弦圓圈自動隱藏
- 使用者可點空弦圓圈切換為靜音（×）

**互動行為：**
- 點格子 → 新增/移除按壓點（toggle）
- 點空弦圓圈 → 空弦 → 靜音 → 空弦（循環切換）
- 清除鈕 → 重置所有弦
- 每次狀態改變立即觸發和弦識別

**觸控區域：** 所有可互動元素最小 44×44pt。

---

### `<ChordResult />` 和弦結果

```
┌──────────────────────────────┐
│  Am/E                        │  ← 主和弦名稱（大字）
│  也可能是 C6/E、Am7/E        │  ← 備選候選（小字）
├──────────────────────────────┤
│  組成音   E  A  C            │  ← pill 樣式音名標籤
│  音程     根音 小三度 完全五度│  ← 對應音程標示
└──────────────────────────────┘
```

每次更新有 fade + slide-up 入場動畫（150ms）。

---

### `<PianoKeyboard />` 鋼琴鍵盤

唯讀，顯示和弦組成音對應的鋼琴鍵位。

**顯示規則：**
- 固定顯示兩個八度，頭尾都是 C（例如 C3–C5）
- 自動選擇最能涵蓋組成音的八度範圍
- 八度切換只在和弦識別結果變更時觸發（不跟著每次點格子跳動）
- 對應鍵用主色高亮

**Phase 1：** 純顯示，不可互動。
**Phase 2：** 開放點選，點鍵後推薦吉他指型。

## 視覺設計

### 色票

| 用途 | Light | Dark |
|------|-------|------|
| 頁面背景 | `#F8F9FF` | `#0F1117` |
| 卡片背景 | `#FFFFFF` | `#1C1F2E` |
| 主色（靛藍） | `#4F6EF7` | `#6B87FF` |
| 強調色（珊瑚橘） | `#FF6B6B` | `#FF8080` |
| 主要文字 | `#1A1A2E` | `#E8EAFF` |
| 次要文字 | `#8B8FA8` | `#6B6F88` |

### 字型
- 標題：`DM Sans`
- 內文/標籤：`Inter`
- 兩者皆可從 Google Fonts 載入

### 指板細節
- 格線交叉點：小圓點（模擬真實琴衍質感）
- 按壓點：主色填色 + 白色音名文字
- 空弦圓圈：主色描邊，無填色

### 動畫
- 按壓點出現/消失：scale + fade（80ms）
- 和弦結果更新：fade + slide-up（150ms）
- Light/Dark 切換：背景色過渡（200ms）

## 狀態管理

三個元件共用一個 composable `useFretboard.ts`：

```ts
const selectedFrets   // Map<弦index, 格數 | 'open' | 'muted'>
const detectedChord   // ChordResult | null
const clearAll()      // 重置指板
```

Phase 1 不需要 Pinia，composable 已足夠。
